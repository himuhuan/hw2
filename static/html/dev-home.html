选择 Language Server Protect (LSP) 的理由

各个补全方案的对比
名称 补全效率 是否自动生成索引 外部环境依赖 本身占用体积
IDE 高, 取决于 IDE 是 否 很大
编辑器 + ctags 传统方案 低 否, 每次 make 后需要重新生成 是 小
LSP 高 是 是, 通常依赖 Nodejs/C++ 后端 小
